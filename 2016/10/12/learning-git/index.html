Git 快速掌握

Posted on 2016-10-12   |   In 工具   |   评论

Git管理的是修改，而不是文件。

1.基本配置

  • 创建版本库，登陆github,create new repository, 版本库命名（不要中文） 勾选上Initialize this repository with a README.
  • 在本地磁盘目录下新建文件夹存放git本地文件。
    git init
Initialized empty Git repository in E:gitFiles/.git/
    初始化后会有.git文件，通过ls -ah命令显示
  • 从远程库克隆到本地库
    git clone git@github.com:VickyXZhang/learning-git.git
    此时，在gitFiles文件夹下就同步了一个learning-git文件夹，此目录下有一个READ.md文件

2.常用操作命令

  1. 文件添加

    • 在本地文件夹learning-git中添加一个文件a.txt, 编辑文本’this is a.’
    • 将文件从工作区添加到暂存区，git add
      git add

  2. 文件提交到版本库

    • 从暂存区提交到版本库，git commit -m “message”
    • 可以多次add, 一次提交
      git commit -m "1st commited"
  3. 文件撤销
    • 撤销工作区中的修改 git checkout –filename
      编辑文本，添加一行”this is aa”.
      撤销此处修改
      git checkout -- a.txt
    • 撤销暂存区中的修改,分为两步,现将暂存区的文件回退到工作区 git reset HEAD filename, 然后再使用上面的方法撤销更改.
      执行git add.
      编辑文本，添加一行”this is aa.”.
      撤销此处修改,此时处于暂存区, 需要执行两步来撤销修改
      git reset HEAD a.txt
git checkout -- a.txt
    • 撤销版本库中的修改, git reset - -hard HEAD^.
      执行git add, git commit, 此时处于版本库中;
      编辑文本, 添加一行”this is aaaa.”. git add, git commit
      撤销此处修改
      git reset --hard HEAD^
  4. 文件删除
    • 在工作区直接执行rm 进行删除, 对暂存区版本库没有任何改变，文件依旧存在
    • 执行 git rm 删除文件
    • commit 到版本库.
      git rm 删除文件会影响到工作区以及暂存区，提交以后会直接影响到版本库，但不影响提交以前版本库的目录结构。

3.分支

  1. 创建分支，并切换分支
    git checkout -b vicky
  2. 编辑文本，添加一行”branch vicky created”.
    add, commit该条修改.
  3. 切换回到master
    git checkout master
  4. 合并分支到master上，这样实现在分支上修改， 结果提交到master, 并不直接修改master
    git merge vicky
  5. 删除分支
    git branch -d vicky
  6. 查看分支结构
    git branch

4.远程操作

  1. git push 将本地文件推送到远程
    git push origin master
  2. git fetch 从远程获取最新版本到本地，不会自动merge
    git fetch origin master
  3. git pull 从远程获取最新版本并merge到本地
    git pull origin master
